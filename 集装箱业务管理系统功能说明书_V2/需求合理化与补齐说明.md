# 需求合理化与补齐说明

> 编制日期：2026-07-16  
> 对照文档：《集装箱业务管理系统功能说明书_V2》  
> 本轮范围：P0 + P1（系统内可闭环；真外联仍为可选）

## 1. 行业对齐摘要

| 来源 | 对本系统的要求 | 本轮落地 |
| :--- | :--- | :--- |
| IICL / UCIRC / BIC 交接检验 | 提还箱须有箱况检查与异常分流 | P1 提箱前箱况确认；异常可挂修箱通知 |
| IICL TB002 EDI（RELEAS/GATEIN/OUT） | 放箱/还箱/账单可追溯；确认后有出站记录 | 进出场关联单号已有；新增 `outbound_events` 账单推送队列 |
| EIR / 预约惯例 | 通知含箱号、司机、车牌、ETA；工作窗与提前量 | 预约工作时段强校验；还箱预约 ≥24h；通知文案对齐 |
| GB/T 16561 设备交接单、GB/T 11601 | 交接凭证与堆场变更同步库存 | 订单/调运执行中可改堆场并调整库存分桶 |
| 国内空箱调运惯例 | 跨订单还箱、台账可导出 | 跨订单还箱发起 UI；台账/报表 CSV 真导出 |

## 2. 合理化原则

1. 真 SMTP / 真订舱出站 HTTP **不阻塞**闭环：未配置时站内通知 + 出站队列/审计可复核。  
2. 「上传」本轮仅 **附件元数据写库**（无对象存储）。  
3. 「导出」本轮为 **浏览器 CSV 下载**，禁止假 toast。  
4. 不改写 V2 正文；以本说明界定本轮增量。

## 3. 本轮交付清单

### P0
- 调运台账 CSV 导出  
- 承运还箱发起支持跨调运单勾选箱号  
- 订单/调运改提还箱堆场并联动库存  
- 账单最终确认 → `outbound_events`（订舱账单推送）；集成页可标记已投递  
- 预约工作时段 + 还箱提前 24h  
- 还箱证明超 3 天未传提醒  

### P1
- 提箱前箱况检查（通过/异常）  
- `attachments` 附件元数据  
- 库存报表筛选 + CSV  
- 审批链展示名对齐说明书五级  

### 明确不做
真对象存储、强制 SMTP、发票 PDF、代管真实 CODECO、完整 CEDEX。

## 4. 验收

`pnpm test:e2e`、`pnpm check:acl`、`tsc --noEmit` 通过。
