# CentOS 8 生产环境部署指南

面向本项目（Next.js 16 + MySQL 8）在 **CentOS 8** 上的简易生产部署。路径均相对于项目根目录。

---

## 1. 环境要求

| 组件 | 版本要求 |
| --- | --- |
| 操作系统 | CentOS 8（或兼容的 Rocky/AlmaLinux 8） |
| Node.js | ≥ 20 |
| pnpm | ≥ 9 |
| MySQL | ≥ 8.0（字符集 `utf8mb4`） |
| 内存建议 | ≥ 2 GB |

默认应用端口：`3000`（可用环境变量 `PORT` 修改）。

---

## 2. 安装基础软件

以下命令用 `root` 或带 `sudo` 执行。

### 2.1 安装 Node.js 20

```bash
curl -fsSL https://rpm.nodesource.com/setup_20.x | bash -
dnf install -y nodejs
node -v   # 应 ≥ v20
```

### 2.2 安装 pnpm

```bash
npm install -g pnpm
pnpm -v
```

### 2.3 安装 MySQL 8

若服务器尚无 MySQL：

```bash
dnf install -y mysql-server
systemctl enable --now mysqld
mysql_secure_installation
```

创建业务库账号（示例，请改密码）：

```sql
CREATE DATABASE IF NOT EXISTS container_biz
  DEFAULT CHARACTER SET utf8mb4
  DEFAULT COLLATE utf8mb4_unicode_ci;

CREATE USER IF NOT EXISTS 'jzx'@'localhost' IDENTIFIED BY '请改成强密码';
GRANT ALL PRIVILEGES ON container_biz.* TO 'jzx'@'localhost';
FLUSH PRIVILEGES;
```

## 3. 部署代码

```bash
mkdir -p /opt/jizhuangxiang
cd /opt/jizhuangxiang
```

### 方式 A：安装 Git 后拉取（推荐，以后更新方便）

```bash
dnf install -y git
git clone https://github.com/liuyangexe2000/jizhuangxiang.git .
pnpm install
```

### 方式 B：服务器没有 / 不想装 Git（上传压缩包）

在**本机**（已有最新代码的电脑）打包，注意排除依赖与构建产物：

**Windows PowerShell（项目根目录）：**

```powershell
# 生成不含 node_modules / .next / .git 的 zip
Compress-Archive -Path * -DestinationPath ..\jizhuangxiang-deploy.zip -Force
# 若上面会打进 node_modules，改用手动选文件，或先：
# 用资源管理器打包时排除：node_modules、.next、.git、.env*.local
```

更稳妥：本机用 GitHub 网页下载，或：

```powershell
# 若本机有 git：打一份干净源码包
git archive -o ..\jizhuangxiang-deploy.zip HEAD
```

把 `jizhuangxiang-deploy.zip` 传到服务器（`scp` / SFTP / 宝塔上传均可），然后：

```bash
cd /opt/jizhuangxiang
# 假设 zip 已放到 /tmp
dnf install -y unzip   # 若尚未安装
unzip -o /tmp/jizhuangxiang-deploy.zip -d /opt/jizhuangxiang
pnpm install
```

> 上传包里**不要**带本机的 `.env*.local`（避免把开发库密码带到生产）；在服务器按下一节重新写 `.env.production.local`。

之后无论哪种方式，都继续：配置环境变量 → 初始化库 → 构建启动。

## 4. 配置生产环境变量

```bash
cp .env.example .env.production.local
vi .env.production.local
```

**至少配置：**

```bash
DB_HOST=127.0.0.1
DB_PORT=3306
DB_USER=jzx
DB_PASSWORD=请改成强密码
DB_NAME=container_biz
DB_POOL_SIZE=10

# 会话签名密钥（必改，勿用示例值）
APP_SECRET=用下面命令生成

# 对外访问地址（域名或 IP，含协议；HTTPS 上线后改为 https）
APP_BASE_URL=http://你的服务器IP:3000

# 种子用户初始密码（仅初始化脚本使用）
SEED_PASSWORD=Passw0rd!
```

生成密钥：

```bash
openssl rand -base64 32
```

可选（按需取消注释）：`SMTP_*`、`BOOKING_API_URL`、`BOOKING_OUTBOUND_URL`，说明见 `.env.example`。

> 权限建议：`chmod 600 .env.production.local`，勿提交到 Git。

## 5. 初始化数据库（仅首次）

```bash
cd /opt/jizhuangxiang
pnpm db:init
```

作用：建表 + 导入种子数据 + 写入用户密码哈希。

> **警告**：`db:init` 会先清空各业务表再导入，**切勿**在已有生产数据上重复执行。后续升级若只改代码/表结构，请按变更说明做 `ALTER`，不要盲目重跑 `db:init`。

默认账号（密码为 `SEED_PASSWORD`，默认 `Passw0rd!`）：

| 账号 | 角色 |
| --- | --- |
| `admin` | 系统管理员 |
| `zhangwei` | 箱管 |
| 其他种子账号 | 见 `README.md` |

上线后请立刻修改 `admin` 密码。

## 6. 构建与启动

### 6.1 构建

```bash
cd /opt/jizhuangxiang
pnpm build
```

### 6.2 用 systemd 常驻（推荐）

创建 `/etc/systemd/system/jizhuangxiang.service`：

```ini
[Unit]
Description=集装箱业务管理系统
After=network.target mysqld.service

[Service]
Type=simple
User=root
WorkingDirectory=/opt/jizhuangxiang
Environment=NODE_ENV=production
Environment=PORT=3000
ExecStart=/usr/bin/pnpm start
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target
```

> 若 `pnpm` 不在 `/usr/bin/pnpm`，用 `which pnpm` 填真实路径。生产建议改用非 root 用户并授权目录权限。

启用并启动：

```bash
systemctl daemon-reload
systemctl enable --now jizhuangxiang
systemctl status jizhuangxiang
journalctl -u jizhuangxiang -f
```

浏览器访问：`http://服务器IP:3000`。

### 6.3 临时前台启动（仅调试）

```bash
cd /opt/jizhuangxiang
NODE_ENV=production PORT=3000 pnpm start
```

## 7. 防火墙

```bash
firewall-cmd --permanent --add-port=3000/tcp
# 若前面用 Nginx 反代，改为开放 80/443：
# firewall-cmd --permanent --add-service=http
# firewall-cmd --permanent --add-service=https
firewall-cmd --reload
```

云主机还需在安全组放行对应端口。

## 8. （可选）Nginx 反代 + HTTPS

安装 Nginx 后示例配置（`/etc/nginx/conf.d/jizhuangxiang.conf`）：

```nginx
server {
    listen 80;
    server_name your.domain.com;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
```

```bash
nginx -t && systemctl reload nginx
```

申请证书可用 `certbot`。HTTPS 上线后请把 `.env.production.local` 中的 `APP_BASE_URL` 改为 `https://your.domain.com`，并重启应用。生产会话 Cookie 依赖 HTTPS 的 `Secure` 标志。

## 9. 日常更新发布

**有 Git：**

```bash
cd /opt/jizhuangxiang
git pull
pnpm install
pnpm build
systemctl restart jizhuangxiang
```

**无 Git：** 本机重新 `git archive` / 打 zip → 上传覆盖到 `/opt/jizhuangxiang`（勿覆盖服务器上的 `.env.production.local`）→ 再执行：

```bash
cd /opt/jizhuangxiang
pnpm install
pnpm build
systemctl restart jizhuangxiang
```

有数据库结构变更时，按发布说明执行 SQL / 迁移脚本，**不要**默认执行 `pnpm db:init`。

## 10. 上线检查清单

1. `systemctl status jizhuangxiang` 为 `active (running)`。
2. `curl -I http://127.0.0.1:3000/login` 返回 200。
3. 用 `admin` 登录成功，进入仪表盘。
4. 打开「系统管理 · 数据管理中心」，各表有数据条数。
5. 改一条业务数据后刷新仍在，说明已写入 MySQL。
6. 打开「操作日志」能看到写操作记录。
7. 登出后直接访问业务页会跳回登录。

## 11. 常见问题

| 现象 | 处理 |
| --- | --- |
| 能打开页面但数据像演示、刷新丢失 | 未连上 MySQL 或未配置 `DB_*`，应用走了内存后端；检查 `.env.production.local` 与 `mysqld` 状态 |
| 登录后立刻掉线 | 检查 `APP_SECRET` 是否固定；多实例时须共用同一密钥；HTTP/HTTPS 混用也会影响 Cookie |
| `pnpm build` 内存不足 | 临时增大 swap，或在更大内存机器上构建后把 `.next` 一并同步 |
| 外网打不开 | 查 `firewall-cmd`、云安全组、以及服务是否只监听了 `127.0.0.1`（`pnpm start` 默认 `0.0.0.0`） |
| 端口被占用 | `ss -lntp \| grep 3000`，改 `PORT` 或停掉占用进程 |
