## 1. 行业对齐摘要

| 来源 | 对本系统的要求 | 本轮落地 |
| :--- | :--- | :--- |
| IICL / UCIRC / BIC 交接检验 | 提还箱须有箱况检查与异常分流 | P1 提箱前箱况确认；异常可挂修箱通知；P2 还箱异常默认箱损费出账 |
| IICL TB002 EDI（RELEAS/GATEIN/OUT） | 放箱/还箱/账单可追溯；确认后有出站记录 | 进出场关联真实箱号；`outbound_events` + **HTTP 投递**（`BOOKING_OUTBOUND_URL`） |
| EIR / 预约惯例 | 通知含箱号、司机、车牌、ETA；工作窗与提前量 | 预约工作时段强校验；还箱预约 ≥24h；通知文案对齐 |
| GB/T 16561 设备交接单、GB/T 11601 | 交接凭证与堆场变更同步库存 | 订单/调运执行中可改堆场并调整库存分桶 |
| 国内空箱调运惯例 | 跨订单还箱、台账可导出 | 跨订单还箱发起 UI；台账/报表 CSV 真导出 |

## 2. 合理化原则

1. 真 SMTP / 真订舱出站 HTTP **不阻塞**闭环：未配置时站内通知 + 出站队列/审计可复核；配置 `BOOKING_OUTBOUND_URL`（或默认本地 echo）后可真实 POST。  
2. 「上传」已支持 **本地落盘**（`attachments.storagePath` + `/api/orders/[id]/upload-doc`）；无对象存储亦可闭环。  
3. 「导出」为 **浏览器 CSV 下载**，禁止假 toast。  
4. 不改写 V2 正文；以本说明界定增量交付。

## 3. 本轮交付清单

### P0
- 调运台账 CSV 导出  
- 承运还箱发起支持跨调运单勾选箱号  
- 订单/调运改提还箱堆场并联动库存  
- 账单最终确认 → `outbound_events`（订舱账单推送）；集成页可标记已投递  
- 预约工作时段 + 还箱提前 24h  
- 还箱证明超 3 天未传提醒  

### P1
- 提箱前箱况检查（通过/异常）  
- `attachments` 附件元数据  
- 库存报表筛选 + CSV  
- 审批链展示名对齐说明书五级  

### P2（2026-07-17 已交付）
- **现场执行态（BR-19/20）**：`confirm-pickup` / `confirm-return` 驱动提箱中/已完成；库存与 gate 联动  
- **真实箱号放箱**：现场选在场箱号写入订单 `containerNos`，按箱写出场/进场 gate 并更新主档状态（替换 USEBOX 合成号）  
- **异常半自动出账**：正常收箱超期 →「超期费账单」；箱况异常 →「箱损费账单」+ 修箱；系统参数可配免租天数/日费率/默认箱损费  
- **账单异议闭环（BR-22）**：客户异议 → 箱管调金额 → 再确认  
- **customerId 贯通**：订单/账单可选客户主档 id；生命周期档案优先按 id 匹配  
- **出站真实 HTTP**：`POST /api/outbound/[id]/deliver`、`/api/outbound/flush`；集成页「HTTP 投递 / 批量投递」  
- **客户/箱生命周期档案**：`/config/customers/[id]`、`/inventory/containers/[containerNo]`  
- **随箱资料/还箱证明真上传与下载**；打印同源 iframe 自包含样式  

### 明确不做（仍可选后续）
真对象存储、强制 SMTP、发票 PDF、代管真实 CODECO、完整 CEDEX；预约堆场侧「确认」状态机（原阶段 A）仍可单独立项。
