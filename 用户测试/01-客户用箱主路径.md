## 4. 主路径用例（推荐最短闭环）

### 用例 UT-UB-01：提交 → 箱管确认 → 提箱 → 还箱完成

#### 前置
- 服务已启动；`customer_xa` / `zhangwei` / `Passw0rd!` 可用。

#### 步骤

| # | 角色 | 页面 | 操作 | 期望结果 | 结果 |
| :- | :--- | :--- | :--- | :--- | :--- |
| 1 | R03 | `/login` | 登录 `customer_xa` | 侧栏有用箱申请/我的订单；**无**订单处理 | ☐ |
| 2 | R03 | `/customer/apply` | 填城市/箱型/数量 → **获取用箱价格** → **提交申请** | Toast：等待箱管确认；「我的订单」为 **待确认**；详情无堆场/后台备注 | ☐ |
| 3 | R01 | `/operations/usebox` | 对待确认单点 **确认申请**：选提/还箱堆场，可改单价、填备注 → **确认并出账** | 状态 **已确认**；`releaseDocReady`；用箱账单按成交价生成；提箱堆场库存 `reserved` 增加/`available` 减少（预占） | ☐ |
| 4 | R03 | `/customer/orders` | 查看详情 | 可见堆场、成交价、箱管备注；**无**确认按钮 | ☐ |
| 5 | R03 | `/customer/bills` | 确认用箱账单 | 状态已确认 | ☐ |
| 6 | R03 | `/customer/documents` | **提箱单** / **预约** | 提箱单可打印且含堆场与备注 | ☐ |
| 7 | R03 | 同上 | **上传随箱资料**（原「箱况确认→通过并上传」） | 仅登记附件，订单**仍为已确认**（不越权推进） | ☐ |
| 8 | R04/R06/R01 | `/customer/documents` | 对该订单点 **确认放箱**：验箱通过 | 生成出场 `gate`；提箱堆场库存联动出场；订单进入 **提箱中**，写入 `pickupGateBy/At` | ☐ |
| 9 | R03 | 同上（还箱作业） | **上传证明**（原会驱动完成，现仅存档） | 仅登记附件，订单**仍为提箱中** | ☐ |
| 10 | R04/R06/R01 | 同上 | 对该订单点 **确认收箱**：验箱通过 | 生成进场 `gate`；还箱堆场库存联动进场；订单 **已完成**，写入 `returnGateBy/At` | ☐ |

#### 通过标准
- 客户不可将订单改为已确认，也不可将订单直改为提箱中/已完成（API 均 403）。
- 仅箱管确认后可打印提箱单；执行态迁移须现场（R04/R06，或 R01/R00 代为）确认放箱/收箱驱动。
- 步骤 8/10 的验箱/异常分支、gate 与库存联动细节、租户隔离负向见 [05-提还箱现场执行与异常](./05-提还箱现场执行与异常.md)。

## 5. 分支用例

### UT-UB-02：箱况异常 → 挂修

（须先完成箱管确认）现场确认放箱/收箱时选 **异常（通知+挂修）** → 订单状态不前进，R01 修箱流转至已完工。详见文档 05。

### UT-UB-03：待确认免责取消

R03 在待确认状态 **取消订单** → **已取消**，无取消费。

### UT-UB-04：超时取消（可选）

已确认且截止已过 → 取消 → **超时取消** + 变更费账单；若已在分配堆场预占库存，取消时同步释放 `reserved`。

### UT-UB-05：账单异议 → 箱管调整 → 客户再确认（BR-22）

1. R03 对「待确认」账单点 **异议**，填写原因 → 状态 **有异议**，写入 `disputeReason`，R01 inbox 可见通知。
2. R01 在账单中心对「有异议」账单点 **调整账单**：修改金额、填写调整说明 → 状态重置为 **待确认**，写入 `adjustedBy`，确认截止重算 72h，通知客户。
3. R03 对调整后账单再次 **确认**（或再次异议，循环 1–2）→ 最终状态 **已确认**，可支付。

### UT-UB-06：箱管代客申请（可选）

R01 在用箱申请必选客户后提交 → 出现在订单处理待确认队列。

## 6. 多联公司角色互动

| 角色 | 要点 |
| :--- | :--- |
| R01 | 订单处理确认（预占库存）；代客申请；改堆场；修箱；预约通知；账单调整（BR-22）；可代堆场执行现场确认 |
| R02 | 侧栏无「我的订单 / 订单处理 / 用箱申请」 |
| R03 | 仅提交、上传资料/证明（存档）与账单确认/异议；不可自行确认订单或推进提箱中/已完成 |
| R04/R06 | 现场确认放箱/收箱（`/customer/documents`），驱动 gate + 库存联动（见文档 05） |

## 7. 关键业务规则速查

| 规则 | 表现 |
| :--- | :--- |
| 提交 | 系统报价写入 `quotedUnitPrice`，状态待确认 |
| 确认 | R01 分配堆场、可改 `unitPrice`、写 `adminRemark` 后出账并放行提箱单，并按提箱堆场预占库存（`reserved`↑/`available`↓） |
| BR-03 | 确认后 24h 内免责取消；待确认取消一律免责；已确认取消同步释放预占库存 |
| BR-05/06 | 预约与还箱证明（证明仅存档，不驱动订单状态） |
| BR-19（新） | 提箱中/已完成须由现场 `confirm-pickup`/`confirm-return` 驱动，客户直改 403 |
| BR-22（新） | 账单异议→箱管调整（重置待确认+72h）→客户再确认，循环至确认/支付/超时默认确认 |
